Build an app where users can:

create and manage customers
create and manage products
create and manage orders
track payments
view sales reports

This is a strong learning project because it naturally uses CRUD, references, embedding, indexes, and aggregation.
Suggested collections
1. customers
Store customer profile data.

_id
name
email
phone
addresses (embedded array)

2. products
Store product catalog.

_id
name
category
price
stock
supplierId (reference if you add suppliers)

3. orders
Main transaction collection.

_id
customerId (reference to customers)
items (embedded array of ordered products)

productId
productName
quantity
unitPrice


status
orderDate
shippingAddress (embedded snapshot)
totalAmount

4. payments
Store payment details separately.

_id
orderId (reference to orders)
method
paidAmount
paymentDate
status

Why this design works
Use embedding where data belongs tightly to the parent and should be read together:

customer addresses inside customers
order items inside orders
shipping address snapshot inside orders

Use references where data is shared or grows independently:

orders.customerId
payments.orderId
items.productId

That gives you both major MongoDB patterns in one app.
CRUD features to build
Customer CRUD

create customer
update phone/email/address
view customer profile
delete customer if no active orders

Product CRUD

add product
update price/stock/category
view product list
delete inactive product

Order CRUD

create order for a customer
add/remove items
update status: Pending, Paid, Shipped, Delivered, Cancelled
view order details
cancel order

Payment CRUD

record payment
update payment status
view payments by order
refund or mark failed payment

MongoDB concepts you will practice
1. References
Use customerId, orderId, and productId to connect collections.
2. Embedding
Store order items and addresses inside parent documents.
3. Aggregation
This use case is excellent for aggregation practice.
Example reports:

total sales by month
total sales by city
top 5 products sold
revenue by category
orders by status
customer lifetime value
unpaid orders report

Common stages you’ll use:

$match
$lookup
$unwind
$group
$project
$sort
$limit

4. Indexes
Add indexes on:

customers.email
products.category
orders.customerId
orders.orderDate
orders.status
payments.orderId

5. Validation / schema discipline
Even with MongoDB flexibility, define clear field rules in Java models and collection validation.
